모바일 GitHub 업로드용 버전입니다.

assets 폴더를 사용하지 않습니다.
사진 파일을 포함한 모든 파일을 저장소 루트에 한꺼번에 업로드하세요.

필수 절차
1. 저장소에서 기존 파일을 모두 덮어씁니다.
2. 이 ZIP 압축을 풉니다.
3. 압축을 푼 폴더 안의 파일을 전부 선택해 업로드합니다.
4. Commit changes를 누릅니다.
5. 배포 후 기존 설치 앱을 삭제합니다.
6. Chrome의 사이트 설정에서 저장공간/캐시를 삭제합니다.
7. GitHub Pages 주소를 다시 열고 앱을 재설치합니다.

이 버전의 HTML은 assets/ 경로를 사용하지 않으므로
사진 파일들이 저장소 첫 화면(root)에 있어도 정상 표시됩니다.
